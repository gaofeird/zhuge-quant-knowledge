## 二、论文核心要点

### 2.1 TradingAgents

**核心创新**：
1. **多智能体协作架构**：模拟真实交易公司
   - 分析师团队：基本面、情绪、新闻、技术
   - 研究员团队：多空辩论
   - 交易员+风控团队+基金经理

2. **混合通信协议**：
   - 结构化输出：精准、无丢失
   - 自然语言对话：灵活性、深度推理

3. **分层LLM策略**：
   - 快思考模型：数据检索、指标计算
   - 深思考模型：基本面分析、交易决策

**回测结果**：
- 累计收益：23-27%
- 夏普比率：5.6-8.2
- 最大回撤：<3%

### 2.2 Qlib

**核心设计**：
1. **AI导向框架**：模块化、支持AI解决方案
2. **高性能数据基础设施**：
   - 时序扁平文件数据库
   - 表达式引擎
3. **Alpha158因子集**：158个量化因子

**技术亮点**：
- 数据处理性能远超HDF5/MySQL/MongoDB
- 表达式引擎支持公式化因子计算

### 2.3 FinRL

**核心架构**：
1. **三层架构**：
   - 应用层：股票交易、组合分配、加密货币
   - 智能体层：PPO、A2C、DDPG、TD3、SAC
   - 环境层：OpenAI Gym风格

2. **MDP建模**：
   - 状态：账户+价格+持仓+技术指标
   - 动作：离散/连续
   - 奖励：账户收益

3. **交易约束**：
   - 交易成本、滑点
   - T+1制度

## 五、核心设计模式

### 5.1 事件驱动模式

```
来源: VeighNa EventEngine

优势:
- 组件解耦
- 异步处理
- 可扩展性

在我们的应用:
- 数据接收 → 事件 → 策略处理
- 信号检测 → 事件 → 风控检查
```

### 5.2 发布-订阅模式

```
来源: TradingAgents Graph

优势:
- 松耦合
- 动态订阅
- 广播通信

在我们的应用:
- 分析师发布报告
- 策略订阅信号
- 风控订阅订单
```

### 5.3 策略模式

```
来源: FinRL Environment

优势:
- 算法可替换
- 参数可配置
- 易于测试

在我们的应用:
- 策略抽象基类
- 多种内置策略
- 支持自定义策略
```

### 5.4 观察者模式

```
来源: 多框架共用

优势:
- 一对多依赖
- 自动通知
- 状态同步

在我们的应用:
- 持仓变化 → 通知UI
- 订单状态 → 通知日志
```
